# 银行核心系统架构

## 核心系统定义

银行核心系统（Core Banking）是银行最基础的业务系统，处理存款、贷款、结算等核心业务。

## 架构演进

### 1.0 集中式架构

- 大型机/小型机
- 集中式数据库
- 垂直扩展

### 2.0 分布式架构

- X86服务器集群
- 分库分表
- 水平扩展

### 3.0 云原生架构

- 容器化部署
- 微服务架构
- DevOps自动化

## 核心功能模块

| 模块 | 功能 |
|------|------|
| 客户管理 | 客户信息、关系管理 |
| 账户管理 | 账户开立、变更、关闭 |
| 账务处理 | 存款、贷款、利息计算 |
| 清算核算 | 日终批量、账务核对 |
| 交易路由 | 交易分发、处理 |

## 数据架构

- **ODS**：贴源层，原始数据
- **数据仓库**：整合分析
- **数据湖**：非结构化存储
- **实时数据**：流处理计算

## 监管报送

- 1104报表
- 人行金融统计
- 反洗钱报送
- 信用报送
